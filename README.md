# Customer Categorization — ML Production System

A full-stack machine learning application that predicts customer clusters based on demographic and spending data.

## 🏗️ Architecture

```
Customer-Categorization---ML-Project/
│
├── notebooks/               # Original Jupyter notebooks (EDA, Clustering, Classification)
├── models/                  # Saved trained models (generated by save_models.py)
│   ├── model.pkl            # Logistic Regression classifier pipeline
│   ├── pca.pkl              # Fitted PCA model
│   └── preprocessor.pkl     # Fitted preprocessing pipeline
│
├── app/                     # FastAPI backend
│   ├── __init__.py
│   ├── main.py              # API endpoints
│   ├── database.py          # MongoDB integration
│   └── schema.py            # Pydantic request/response models
│
├── frontend/                # Web UI
│   ├── index.html
│   ├── style.css
│   └── script.js
│
├── save_models.py           # Script to train & save models
├── requirements.txt         # Python dependencies
├── Dockerfile               # Docker configuration
├── .dockerignore
└── README.md
```

## 🎯 Features

- **ML Pipeline**: Preprocessing → PCA → KMeans Clustering → Logistic Regression Classification
- **REST API**: FastAPI with automatic documentation at `/docs`
- **Database**: MongoDB integration for storing predictions with timestamps
- **Frontend**: Premium dark-theme glassmorphism UI
- **Docker**: Ready for containerized deployment

## 🔌 API Endpoints

| Method | Endpoint    | Description                          |
|--------|-------------|--------------------------------------|
| GET    | `/`         | Welcome message & API info           |
| GET    | `/ui`       | Serve the frontend UI                |
| GET    | `/docs`     | Interactive API documentation        |
| POST   | `/predict`  | Predict customer cluster             |

### POST `/predict` — Example

**Request:**
```json
{
    "Age": 30,
    "Income": 50000,
    "Total_Spending": 2000,
    "Children": 1,
    "Education": 2
}
```

**Response:**
```json
{
    "cluster": 1,
    "category": "Medium Value Customer"
}
```

### Cluster Labels
| Cluster | Category              |
|---------|-----------------------|
| 0       | Low Value Customer    |
| 1       | Medium Value Customer |
| 2       | High Value Customer   |

## 🚀 Getting Started

### Prerequisites
- Python 3.9+
- MongoDB (optional — app works without it)
- Docker (optional — for containerized deployment)

### 1. Install Dependencies
```bash
pip install -r requirements.txt
```

### 2. Train & Save Models
```bash
python save_models.py
```
This reads the dataset, trains the pipeline, and saves pickle files to `models/`.

### 3. Run the Application
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 4. Open in Browser
- **API Docs**: http://localhost:8000/docs
- **Frontend UI**: http://localhost:8000/ui

## 🐳 Docker Deployment

### Build the Image
```bash
docker build -t customer-categorizer .
```

### Run the Container
```bash
docker run -p 8000:8000 customer-categorizer
```

> **Note**: If you want MongoDB inside Docker, use a `docker-compose.yml` or connect to an external MongoDB instance by setting the `MONGO_URI` environment variable:
> ```bash
> docker run -p 8000:8000 -e MONGO_URI="mongodb://host.docker.internal:27017" customer-categorizer
> ```
